Mybatis | SqlMapConfig

发表于2021-08-23|更新于2022-03-10|笔记

properties（属性）

需求

　　将数据库连接参数配置在 db.properties 中，在 SqlMapConfig.xml 中通过 properties 加载 db.properties 的属性值，在 SqlMapConfig.xml 中不需要对数据库连接硬编码

注意

　　mybatis 按下面的顺序加载属性

  • 在 properties 元素内定义的属性首先被读取
  • 然后读取 properties 元素中 resource 或 url 加载的属性，它会覆盖已读取的属性
  • 最后读取 parameterType 传递的属性，它会覆盖一读取的同名属性

　　因此，通过 parameterType 传递的属性具有最高优先级，resource 或 url 加载的属性次之，最低优先级的是 properties 元素体内定义的属性

建议

　　直接属性值定义在properties文件中

useGeneratedKeys 允许JDBC支持生成的密钥，兼容的驱动程序是必须的，此设置强制生成的键被使用，如果设置为true，一些驱动会不兼容性，但仍然可以工作 true | false FALSE

safeRowBoundsEnabled 允许使用嵌套的语句RowBounds true | false FALSE
mapUnderscoreToCamelCase 从经典的数据库列名A_COLUMN启用自动映射到骆驼表示的经典Java属性名aColumn true | false FALSE
localCacheScope MyBatis的使用本地缓存，以防止循环引用，并加快反复嵌套查询，默认情况下（SESSION）会话期间执行的所有查询缓存。如果localCacheScope=STATMENT本地会话将被用于语句的执行，只是没有将数据共享之间的两个不同的调用相同的SqlSession SESSION、STATEMENT SESSION
dbcTypeForNull 指定为控制时，没有特定的JDBC类型的参数的JDBC类型，有些驱动需要指定列的JDBC类型，但其他像NULL、VARCHAR或OTHER的工作与通用值 JdbcType enumeration、Most common are: NULL、VARCHAR and OTHER OTHER
lazyLoadTringerMethods 指定触发延迟加载的对象的方法 A method name list separated by commas quals、clone、hashCode、toString
defaultScriptingLanguage 指定所使用的语言默认为动态SQL生成 A type alias or fully qualified class name ore.apache.itbatis.scripting.xmltags.XMLDynamicLanguageDriver
callSettersOnNulls 指定如果setter方法或地图的put方法时，将调用检索到的值是null，它是有用的，当你依靠Map.keySet()或null初始化，注意冤狱（如整型、布尔等）不会被设置为null true | false FALSE
logPrefix 指定的前缀字串，MyBatis将会增加记录器的名称 Any String Not set
logImpl 指定MyBatis的日子实现使用，如果此设置是不存在的记录的实施将自动查找 SLF4J | LOG4J | LOG4J2 | JDK_LOGGING | COMMONS_LOGGING | STDOUT_LOGGING | NO_LOGGING Not set
proxyFactory 指定代理工具，Mybatis将会使用创建懒加载能力的对象 CGLIB | JAVASSIST CGLIB

需求

　　在mapper.xml中，定义很多的statement，statement需要parameterType指定输入参数的类型、需要resultType指定输出结果的映射类型，如果在指定类型时输入类型全路径，不方便进行开发，可以针对parameterType或resultType指定的类型定义一些别名，在mapper.xml中通过别名定义，方便开发
